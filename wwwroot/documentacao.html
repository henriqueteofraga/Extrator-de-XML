🚀 O que é este Extrator?

Este sistema foi desenvolvido para ser uma ferramenta que permite extrair os XMLs dos cupons diretamente do banco de dados, sem a necessidade de utilizar o Gestão, proporcionando mais agilidade ao processo.

Diferentemente da ferramenta interna do Gestão, "Baixar XML", que realiza o download de um arquivo por vez, este motor agrupa as vendas em lotes de 500 registros, descompacta os pacotes e entrega os arquivos prontos no diretório escolhido.

OBS: Por enquanto, o sistema não está configurado os bancos de dados dos clientes SkyOne, com exceção do Curitibanos

🧠 Como o sistema funciona?

O sistema trabalha em um modelo de Rede Distribuída dividido em três partes:

  • Interface Web: É a página que você está visualizando. Ela pode ser aberta de qualquer computador ou celular conectado à rede interna da empresa.
  • Banco de Configurações Central: Fica armazenado permanentemente no servidor da API. É onde ficam guardadas as regras fiscais estruturadas. Em casos de mais detalhes a respeito, pelo Henrique Fraga.
  • Servidor de banco de dados: São as máquinas que possuem os bancos de dados Gestao e GestaoXML (onde os XMLs estão guardados).
Informação Importante: Quando você inicia uma extração, a API busca a inteligência do código SQL no Banco Central, conecta-se no servidor de banco de dados que você digitou, processa as informações lá dentro e grava os arquivos na pasta selecionada.

🛠️ Como Utilizar o Extrator?

Siga este passo a passo sequencial para realizar uma extração sem erros:

  1. Identifique o Servidor de banco de Dados: No campo Servidor (IP ou Nome), insira o computador da rede de onde deseja extrair os XMLs. Insira o usuário e senha do SQL Server daquela máquina e clique em "Conectar e Listar Bancos".
  2. Selecione o Banco de Dados: O sistema listará na lista suspensa todos os bancos de movimentação daquela máquina (ex: GestaoXML_13986009000165). Escolha o desejado.
  3. Defina o Período: Escolha a Data Inicial e a Data Final do período que o cliente ou a contabilidade solicitou.
  4. Escolha a Regra Fiscal: Selecione o filtro de extração desejado (ex: Vendas Cesta Básica). O sistema filtrará os XMLs para trazer apenas os cupons que se enquadrem na regra criada.
  5. Escolha a Pasta de Destino: Por padrão, este campo já vem preenchido com um diretório padrão de rede para centralizar e organizar as extrações automaticamente. Seja utilizando o caminho padrão ou digitando um novo, lembre-se de que o diretório deve obrigatoriamente ser um caminho de rede visível para o servidor (no formato UNC, ex: \\IP_OU_NOME_DA_MAQUINA\PastaCompartilhada) e a máquina onde a API está rodando precisa ter permissões totais de leitura e escrita liberadas nessa pasta.
    Dica: Antes de iniciar a exportação, acesse a pasta padrão (ou a pasta que decidiu utilizar) e crie uma subpasta para a sua demanda. Isso evitará misturar XML's em caso de usos simultâneos
    ⚠️ Por que não posso digitar o caminho local da minha própria máquina (ex: C:\MinhaPasta)? Por questões estritas de segurança, os navegadores web (Chrome, Edge, etc.) são completamente bloqueados de acessar os arquivos locais do seu computador pessoal. Além disso, quem realiza a gravação física dos XMLs é o servidor da API. Se você digitar C:\MinhasNotas, o sistema vai tentar caçar essa pasta dentro do disco rígido do servidor, e não no seu, o que causará uma falha no processo. Por isso, para salvar na sua máquina, use sempre o caminho de rede compartilhado (\\IP...). Se preferir salvar direto em um disco do próprio servidor da API, você também pode clicar no botão 📁 "Procurar..." para abrir o seletor visual.
  6. Inicie o Processo: Clique no botão verde "Iniciar Extração em Lotes".

📦 O que esperar dos resultados?

Acompanhamento Visual

Assim que o processo inicia, o painel exibe duas informações em tempo real:

Campo O que significa
Total de Vendas Elegíveis A quantidade exata de cupons encontradas no banco que batem com o período e com a regra escolhida.
Progresso Atual A porcentagem de lotes que já foram completamente processados, baixados e extraídos no disco.

Arquivos Gerados

Ao término do processo, os arquivos XML originais estarão extraídos e organizados de forma limpa, com os nomes originais de emissão, diretamente dentro do diretório informado no campo de destino.

Tratamento de Erros: Se alguma nota do lote estiver corrompida ou o disco ficar cheio, o sistema pulará o registro problemático para não interromper a operação da rede e gravará o motivo detalhado em um arquivo de log chamado erro_extracao.log diretamente dentro da mesma pasta que você definiu como diretório de destino.

➕ Como cadastrar novas Querys?

Se você precisar filtrar os XMLs por uma nova categoria de impostos ou produtos (ex: Vendas de Bebidas ou Produtos de Higiene), você pode clicar no link "Cadastrar Query" na tela inicial.

Para criar uma regra válida, o técnico responsável precisa apenas estruturar um comando SQL que insira os códigos dos produtos desejados dentro de uma tabela temporária chamada #VendasElegiveis, exatamente como no exemplo abaixo:

-- Todas as vendas IF OBJECT_ID(''tempdb..#VendasElegiveis'') IS NOT NULL DROP TABLE #VendasElegiveis; CREATE TABLE #VendasElegiveis (CodigoProduto BIGINT PRIMARY KEY); INSERT INTO #VendasElegiveis (CodigoProduto) select Codigo from Gestao.dbo.Vendas';

Ao gravar, a regra fica disponível imediatamente para todos os usuários do sistema na rede, sem necessidade de atualizar ou reiniciar o programa.
